## 二、业务分析

### 2.1 消除逻辑

```
核心算法：三消匹配检测
━━━━━━━━━━━━━━━━━━━━━━━━━

1. 横向检测
   遍历每行 → 连续相同元素 ≥ 3 → 标记消除

2. 纵向检测
   遍历每列 → 连续相同元素 ≥ 3 → 标记消除

3. 特殊消除（L型/T型/4连/5连）
   检测匹配形状 → 生成特殊元素（炸弹/彩虹球）

4. 连锁反应
   消除 → 元素下落 → 填充新元素 → 再次检测 → 递归执行
```

**关键代码逻辑**：

```javascript
// 查找匹配
function findMatches(board) {
  const matches = new Set();
  
  // 横向
  for (let r = 0; r < 8; r++) {
    for (let c = 0; c < 6; c++) {
      if (board[r][c] === board[r][c+1] && board[r][c] === board[r][c+2]) {
        matches.add(`${r},${c}`);
        matches.add(`${r},${c+1}`);
        matches.add(`${r},${c+2}`);
      }
    }
  }
  
  // 纵向（同理）
  return Array.from(matches).map(s => {
    const [r, c] = s.split(',').map(Number);
    return { row: r, col: c };
  });
}

// 下落填充
function fillBoard(board) {
  for (let col = 0; col < 8; col++) {
    let empty = 0;
    // 从下往上遍历，空位上移
    for (let row = 7; row >= 0; row--) {
      if (board[row][col] === null) {
        empty++;
      } else if (empty > 0) {
        board[row + empty][col] = board[row][col];
        board[row][col] = null;
      }
    }
    // 顶部填充新元素
    for (let row = 0; row < empty; row++) {
      board[row][col] = randomElement();
    }
  }
}
```

### 2.2 得分规则

```
得分公式：
基础分 = 消除数量 × 10
连击加成 = 基础分 × 连击倍数
最终得分 = 基础分 + (基础分 × 连击数)

示例：
  第1次消除 3个 = 30分
  第2次连锁 5个 = 50 × 2 = 100分
  第3次连锁 4个 = 40 × 3 = 120分
  本轮总计 = 30 + 100 + 120 = 250分
```

## 三、优化建议

### 3.1 游戏流畅度

| 问题 | 解决方案 | 优先级 |
|-----|---------|-------|
| **卡顿** | 使用 `transform` 代替 `top/left` | P0 |
| **动画掉帧** | 使用 `will-change: transform` | P1 |
| **频繁重绘** | 减少 `innerHTML` 重写 | P1 |
| **GC 压力** | 对象池复用 DOM 节点 | P2 |

```css
/* 优化动画性能 */
.cell {
  will-change: transform, opacity;
  transform: translateZ(0); /* 开启硬件加速 */
}
```

### 3.2 画面体验

| 优化点 | 方案 | 效果 |
|-------|------|------|
| **消除反馈** | 爆炸粒子 + 震动 | 打击感 |
| **连击提示** | 大字 + 缩放动画 | 成就感 |
| **背景单调** | 动态祥云/花瓣飘落 | 氛围感 |
| **字体渲染** | 思源黑体清晰 | 舒适阅读 |
| **响应式** | 锁定 500px 最大宽度 | 居中美观 |

### 3.3 动画优化

```javascript
// 推荐：使用 CSS class 触发动画
.cell.matched {
  animation: matchAnim 0.3s ease forwards;
}

// 不推荐：JS 逐帧动画（性能差）
function animate() {
  element.style.top = pos + 'px';
  requestAnimationFrame(animate);
}

// 正确做法：创建临时 DOM 动画后销毁
function showCombo(count) {
  const el = document.createElement('div');
  el.className = 'combo-display';
  el.textContent = `${count}连击!`;
  document.body.appendChild(el);
  setTimeout(() => el.remove(), 800); // 及时销毁
}
```

### 4.3 Bug 预防

| 常见 Bug | 预防措施 |
|---------|---------|
| **初始死局** | 生成后检测可消除，无则重生成 |
| **无限连锁** | 设置最大连锁次数上限 |
| **点击穿透** | 操作时 `isProcessing` 锁 |
| **存储溢出** | 定期清理旧数据，限制排行榜数量 |
| **内存泄漏** | 动画元素及时 `remove()` |
| **ios 点击延迟** | `touch-action: manipulation` |

```javascript
// 死局检测与预防
function ensurePlayable(board) {
  while (hasMatches(board)) {
    removeMatches(board);
    fillBoard(board);
  }
  
  // 还要确保有可移动的空间
  if (!hasPossibleMoves(board)) {
    // 随机打乱重试
    return shuffleBoard(board);
  }
  return board;
}
```
