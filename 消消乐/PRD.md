# 国风消消乐小游戏产品需求文档

---

## 一、产品概述

### 1.1 产品定位

国风消消乐是一款融合中国传统美学元素的经典三消休闲游戏。游戏以水墨画、传统纹样等中式美学为视觉设计核心，采用暖色调配色方案，营造喜庆、温馨的游戏氛围。玩家通过点击或拖拽交换相邻元素，消除三个或更多相同图案，达成关卡目标，体验轻松愉悦的游戏乐趣。

本产品定位为轻量化网页游戏，支持本地直接打开运行，同时预留云端扩展接口，可根据需求部署实现多人在线功能。目标用户为休闲玩家、学生群体、国风文化爱好者及具有社交需求的年轻用户。

### 1.2 产品愿景

打造一款具有鲜明中国文化特色的休闲三消游戏，让玩家在游戏过程中感受传统文化的魅力，同时满足碎片化时间的娱乐需求。

---

## 二、目标用户

### 2.1 用户画像

本产品的核心目标用户可分为以下四类：

第一类为休闲玩家，主要特征为碎片时间较多，追求轻松解压的娱乐方式，核心需求是操作简单、上手快、无需长时间投入，这类用户占比约为总用户量的百分之四十。

第二类为学生群体，主要特征为学业压力较大，偏好轻量化娱乐方式，核心需求是界面美观、节奏明快、可离线游玩，这类用户占比约为总用户量的百分之二十五。

第三类为国风爱好者，主要特征为喜爱传统文化元素，核心需求是水墨意境、文化内涵、视觉享受，这类用户占比约为总用户量的百分之二十。

第四类为社交需求用户，主要特征为喜欢与朋友互动竞技，核心需求是排行榜、多人在线、互动性，这类用户占比约为总用户量的百分之十五。

### 2.2 用户需求分析

用户在游戏过程中的核心需求包括简单易上手的操作方式、视觉愉悦的游戏体验、即开即玩无需下载的便捷性，以及成就感和社交互动。延伸需求则包括丰富的关卡内容、成就系统和收集要素、与朋友比拼的社交需求，以及个性化定制。

## 三、核心玩法

### 3.1 基础规则

#### 3.1.1 三消机制

游戏采用经典的三消玩法设计。棋盘规格为八乘八的标准网格，玩家通过点击或拖拽交换相邻元素的位置。当三个或以上相同元素在水平或垂直方向连成一线时，即可触发消除。消除后上方元素下落，新元素从顶部填充，可能触发连锁消除。

#### 3.1.2 操作方式

游戏支持两种操作方式。点击操作模式下，玩家点击第一个元素将其选中，再点击相邻元素进行交换。拖拽操作模式下，玩家按住第一个元素并拖拽至相邻元素位置完成交换。两种操作方式均可完成游戏，建议保留两种方式以适应不同玩家的操作习惯。

#### 3.1.3 关卡目标

游戏设有得分关和步数关两种关卡类型。得分关的目标是在限定步数内达成指定分数，通关条件为累计分数大于等于目标分数。步数关的目标是在限定步数内达成目标，同样采用累计分数判定通关。

### 3.2 元素体系

#### 3.2.1 基础元素

游戏包含六种基础元素，分别对应六个中国传统符号。灯笼元素对应 unicode 字符 u+1F3EE，文化寓意为喜庆吉祥，视觉特征为红色宫灯造型，主色调为朱红色。玉佩元素对应 unicode 字符 u+1F000，文化寓意为美玉无瑕，视觉特征为圆形玉佩图案，主色调为翠绿色。瓷器元素对应 unicode 字符 u+1F3FA，文化寓意为国粹精华，视觉特征为青花瓷瓶形态，主色调为青蓝色。铜钱元素对应 unicode 字符 u+1F4B0，文化寓意为财源滚滚，视觉特征为圆形方孔铜钱，主色调为金黄色。祥云元素对应 unicode 字符 u+2601FEOF，文化寓意为吉祥如意，视觉特征为卷云纹样，主色调为奶白色。梅花元素对应 unicode 字符 u+1F338，文化寓意为坚韧不拔，视觉特征为梅花枝头绽放，主色调为粉红色。

#### 3.2.2 特殊元素

除基础元素外，游戏还设计了两类特殊元素以增加策略性和爽快感。炸弹元素的触发条件为四连消除，特殊效果为消除周围三乘三区域内的所有元素。彩虹球元素的触发条件为五连消除，特殊效果为消除场上所有指定颜色的元素。

### 3.3 难度设计

#### 3.3.1 关卡难度曲线

游戏共设计十个关卡，难度呈阶梯式递增。第一关目标分数为一千分，步数限制为二十五步，仅出现四种元素类型。第二关目标分数为一千五百分，步数限制为二十五步。第三关目标分数为二千分，步数限制为二十四步。第四关目标分数为二千五百分，步数限制为二十三步。第五关至第十关的目标分数和步数限制逐步提高，至第十关目标分数达到八千分，步数限制为十八步。

#### 3.3.2 难度参数配置

关卡难度通过目标分数、步数限制和元素种类数量三个参数进行配置。入门阶段关卡目标分数较低、步数较多、元素种类较少，让玩家熟悉游戏操作。进阶阶段逐步增加目标分数和难度，帮助玩家建立成就感和挑战性。

## 四、功能模块

### 4.1 游戏核心模块

游戏核心模块包含以下功能要点：

棋盘初始化功能负责生成随机元素分布，确保初始状态不存在可消除的匹配项，同时保证棋盘存在可移动的空间。实现方式为随机算法配合可消除检测，若初始存在匹配则重新生成，若无有效移动则打乱重排。

元素交换功能负责处理玩家点击或拖拽操作，完成相邻元素的交换并触发匹配检测。实现方式为触摸和鼠标事件监听，配合 CSS 动画实现交换过渡效果。

消除检测功能负责检测并标记需要消除的元素。实现方式为遍历算法，检测水平方向和垂直方向是否存在三个及以上连续相同元素。

元素下落功能负责在消除发生后，上方元素下落填补空位。实现方式为列遍历配合空位计数计算。

新元素填充功能负责在顶部生成新元素填补空位。实现方式为队列生成随机元素。

连锁检测功能负责递归检测连锁消除。实现方式为深度优先搜索，检测消除后是否触发新的匹配。

特殊元素生成功能负责在满足条件时生成特殊元素。实现方式为匹配模式识别，检测四连、五连等特殊形状。

关卡判定功能负责判断胜利、失败或步数耗尽等游戏状态。实现方式为状态机判断。

### 4.2 界面模块

游戏界面包含以下六个主要界面：

启动页负责品牌展示和加载进度显示，采用动画过渡效果。

主菜单提供开始游戏、排行榜、设置等入口，采用点击导航方式。

关卡选择界面显示已解锁和未解锁关卡，采用列表展示配合点击选择。

游戏界面是核心游戏区域，显示计时、分数、步数等关键信息，采用全屏触控交互。

结算界面展示通关提示、星级评价和奖励，采用弹窗和按钮交互。

排行榜界面展示本地和云端排名，采用标签页切换。

设置界面提供音效、音乐、语言等设置选项，采用开关控制。

### 4.3 数据统计模块

数据统计模块包含以下统计项和存储方式：

当前分数存储于内存，用于实时显示本局游戏累计得分。当前步数存储于内存，用于显示剩余可行动步数。关卡进度存储于 localStorage，用于记录已通过最高关卡。总游戏次数存储于 localStorage，用于记录历史累计游戏局数。最高分存储于 localStorage，用于记录历史最高单局得分。

### 4.4 社交互动模块

#### 4.4.1 本地排行榜

本地排行榜记录玩家前十分最高分记录，显示排名、玩家名、得分和关卡信息。支持本地数据管理，包括查看和清空功能。数据存储采用 localStorage，键名为 chineseMatch3_leaderboard。

#### 4.4.2 云端排行榜

云端排行榜连接云端服务获取全服排名，支持实时更新和跨设备同步。显示玩家昵称、得分和关卡信息。实现方式为 RESTful API 调用，数据存储于云端数据库。

#### 4.4.3 多人在线系统

多人在线系统支持创建和加入游戏房间，房间人数上限为三十人。系统支持实时同步游戏进度、观战模式和竞速模式。房间内提供聊天功能，采用预设短语和自由输入两种方式。实现方式为 WebSocket 长连接，服务器端使用 Socket.io 框架。

### 4.5 音效与特效模块

#### 4.5.1 音效系统

游戏音效采用 Web Audio API 实现，无需外部音频文件。背景音乐根据场景切换，包括主界面、游戏进行中、胜利结算和失败结算四种场景。音效素材采用合成音，包括点击音效、交换音效、消除音效、连锁音效、特殊消除音效、胜利音效和失败音效。

#### 4.5.2 动画特效

消除特效采用 CSS 动画实现元素爆炸和破碎效果。下落特效采用 CSS transition 实现元素下落弹跳效果。连击特效采用 JavaScript 创建 DOM 动画实现连击数字跳动。特殊特效采用 CSS box-shadow 和 filter 实现光效和光晕。界面切换采用 CSS opacity 和 transform 实现淡入淡出和滑动效果。

## 五、界面风格

### 5.1 视觉定位

游戏整体视觉风格定位为水墨意境、喜庆祥和、温暖如春、古典雅致。通过中国传统元素与现代游戏设计的结合，创造独特的视觉体验。

### 5.2 配色方案

#### 5.2.1 主色调

中国红（色值 C8392D）用于主按钮、重点强调和灯笼元素。暖阳黄（色值 F5A623）用于高亮、得分数字和金币。古典绿（色值 4CAF50）用于玉佩元素和成功状态。青花蓝（色值 3F51B5）用于瓷器元素和背景点缀。胭脂粉（色值 E91E63）用于梅花元素和装饰细节。

#### 5.2.2 辅助色

背景米白（色值 FFF8E1）用于主背景色。宣纸黄（色值 FDF5E6）用于卡片背景。墨黑（色值 2D2D2D）用于文字主色。淡墨灰（色值 757575）用于次要文字。边框金（色值 D4AF37）用于装饰边框。

### 5.3 字体规范

标题字体采用 Ma Shan Zheng 或 STKaiti，字号为三十六至四十八像素，颜色为墨黑色。副标题字体采用 Ma Shan Zheng 或 STKaiti，字号为二十四至二十八像素。正文字体采用 Noto Sans SC 或 sans-serif，字号为十六至十八像素。数字字体采用 Noto Sans SC 或 sans-serif，字号为二十四至四十八像素，颜色为暖阳黄。按钮字体采用 Noto Sans SC 或 sans-serif，字号为十八像素，颜色为白色。

### 5.4 界面布局

主界面布局采用上下结构，顶部为标题区显示游戏名称和副标题，中间为功能按钮区设置开始游戏、排行榜、设置等入口，下方为装饰区添加祥云和灯笼等传统元素。游戏界面布局同样采用上下结构，顶部为信息栏显示关卡、分数和步数，中间为目标进度条显示当前分数与目标分数的比例，核心区域为八乘八游戏棋盘，底部为操作栏提供返回和重玩按钮。

## 六、数据结构

### 6.1 本地存储结构

本地存储采用 localStorage 实现，包含以下三个键名：

chineseMatch3_settings 键存储用户设置信息，包括音效开关状态、音乐开关状态、玩家昵称和当前关卡。数据结构为 JSON 格式，包含 soundEnabled、musicEnabled、nickname 和 currentLevel 四个字段。

chineseMatch3_leaderboard 键存储排行榜数据，包括前十分最高分记录。数据结构为 JSON 格式的数组，每个元素包含 name、score、level 和 date 四个字段。

chineseMatch3_stats 键存储游戏统计数据，包括总游戏次数、最高分数、最高关卡和总得分。数据结构为 JSON 格式，包含 totalGames、highScore、maxLevel 和 totalScore 四个字段。

### 6.2 游戏运行时数据结构

游戏状态存储于内存，包含以下核心字段：

board 字段为八乘八二维数组，存储当前棋盘上各位置的元素信息，每个元素包含 type（元素类型）和 special（特殊元素类型）两个属性。score 字段为当前累计分数。moves 字段为剩余步数。level 字段为当前关卡编号。targetScore 字段为当前关卡目标分数。selectedCell 字段为当前选中的元素位置。isProcessing 字段为游戏是否正在处理消除动画的标志位。combo 字段为当前连击数。

### 6.3 服务器端数据结构

排行榜 API 响应数据结构包含 success 布尔字段和 data 数组字段。房间数据结构包含 roomId（房间标识）、roomName（房间名称）、hostOpenid（房主标识）、players（玩家列表）、maxPlayers（最大人数）、status（房间状态）和 createdAt（创建时间）字段。

## 七、业务流程

### 7.1 游戏主流程

游戏主流程分为五个主要阶段：

第一阶段为启动与初始化，包括页面加载、资源加载和本地存储检查。若检测到无存档则初始化新用户数据。

第二阶段为主菜单界面，玩家可选择开始游戏、进入排行榜或进行设置。

第三阶段为关卡选择，显示已解锁关卡列表，玩家选择关卡后进入游戏。

第四阶段为游戏进行，包括玩家操作、消除处理和关卡判定三个子环节。玩家操作包含元素选择和交换，消除处理包含匹配检测、元素消除、元素下落和新元素填充，关卡判定根据分数和步数判断胜利或失败。

第五阶段为关卡结算，显示得分、星级评价和奖励，提供下一关或重玩选项。

### 7.2 排行榜流程

排行榜流程分为本地排行榜和云端排行榜两个分支。本地排行榜直接读取 localStorage 数据并展示。云端排行榜则通过 API 请求获取服务器数据，成功时展示云端排名，失败时显示错误提示。

### 7.3 多人在线流程

多人在线流程包含房间创建、玩家加入、游戏进行和结算四个阶段。房主创建房间后获得房间号，其他玩家输入房间号加入房间。房间内玩家准备就绪后开始游戏，游戏过程实时同步各玩家进度，游戏结束时展示排名。

## 八、部署方案

### 8.1 前端部署

#### 8.1.1 本地运行

游戏支持直接双击 index.html 文件在浏览器中打开运行。也可使用 Python 的 HTTP 服务器运行，执行 python -m http.server 命令后在浏览器访问本地地址。亦可使用 Node.js 的 serve 工具运行。

#### 8.1.2 云端部署

推荐使用 Vercel、Netlify、GitHub Pages 或阿里云 OSS 等平台进行静态部署。部署方式为将 index.html 文件上传至平台，平台自动生成访问地址。

### 8.2 云端服务部署

#### 8.2.1 服务器要求

云端服务采用 Node.js 部署，需要一台云服务器或使用 Serverless 平台。推荐配置为 CPU 一核、内存 512MB、带宽 1Mbps，可支持三十人同时在线。

#### 8.2.2 服务架构

服务端采用 Express 框架搭建 HTTP API，使用 Socket.io 实现 WebSocket 通信。API 端点包括获取排行榜和提交分数两个接口。WebSocket 处理房间管理、玩家加入、聊天消息和进度同步等事件。

#### 8.2.3 部署平台

可选的部署平台包括 Railway、Render、阿里云和腾讯云。Railway 和 Render 支持 GitHub 仓库自动部署，适合快速上线。阿里云和腾讯云需要手动配置服务器，适合有运维能力的团队。

## 九、验收标准

### 9.1 基础功能验收

三消玩法的验收标准为三个及以上相同元素可消除，交换有动画反馈。关卡系统的验收标准为可正常进入关卡，通关和失败判定正确。分数统计的验收标准为分数累计正确，连击加分生效。步数统计的验收标准为步数正确扣减，步数耗尽时游戏结束。

### 9.2 视觉表现验收

美术风格的验收标准为符合国风、暖色调、水墨意境的要求。动画特效的验收标准为消除、下落和连击有流畅动画。

### 9.3 音效系统验收

背景音乐的验收标准为场景切换时音乐正确切换。音效反馈的验收标准为操作、消除和通关有对应音效。

### 9.4 数据存储验收

本地存储的验收标准为进度、设置和排行榜正确保存和读取。

### 9.5 社交功能验收

本地排行榜的验收标准为显示前十分数据。云端排行榜的验收标准为可获取并展示全服排名（可选功能）。多人功能的验收标准为可创建和加入三十人以内的房间（可选功能）。

### 9.6 性能表现验收

加载速度的验收标准为首屏加载时间小于三秒。运行流畅的验收标准为游戏过程稳定在三十帧每秒以上。
